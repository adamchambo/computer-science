# Data Structures

This section covers ways of organizing, storing, and accessing data efficiently. Each data structure has different performance characteristics, making it suitable for different problems.

## Concepts

- Arrays and dynamic arrays
- Linked lists
- Stacks and queues
- Deques and priority queues
- Hash maps and hash sets
- Trees and binary search trees
- AVL trees
- Heaps
- Tries
- Graphs
- Union-find
- Bloom filters

## What each implementation explores

- How the data structure stores and connects its values
- Its common operations
- The time and space complexity of those operations
- The trade-offs compared with related structures
- A custom implementation in one or more programming languages

## Under the hood

The directories are organized by data structure, not by language. For example, `linked-list/singly-linked-list/` contains implementations of the singly linked list concept. Python is being used first, and implementations in other languages can be added to the same concept directory later.

This structure keeps the computer science concept central while allowing direct comparisons between language implementations.
